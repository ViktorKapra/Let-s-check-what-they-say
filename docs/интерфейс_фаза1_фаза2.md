# Интерфейс между Фаза 1 и Фаза 2 — Формат на данните

**Цел на документа:** Точно описание на какво трябва да произведе Фаза 1, така че Фаза 2 да стартира без проблеми. Включва намерените несъответствия и техните решения.

---

## Очакван изход от Фаза 1

Фаза 2 очаква един единствен файл:

```
data/processed/speeches.csv
```

### Схема на CSV файла

| Колона | Тип | Пример | Описание |
|--------|-----|--------|----------|
| `session_date` | str | `"2026-04-01"` | Дата на заседанието във формат `YYYY-MM-DD` |
| `statement_id` | int | `681213` | Уникален id от strazha.bg |
| `statement_number` | int | `7` | Поредност на изказването в заседанието |
| `mp_id` | int | `1433` | `parlMemberId` от strazha.bg |
| `speaker` | str | `"Йордан Иванов"` | `title` от strazha.bg |
| `position` | str | `"parl-member"` | `"house-speaker"` или `"parl-member"` |
| `party` | str | `"Продължаваме промяната - Демократична България"` | Пълно официално название на партията |
| `text` | str | `"уважаеми господин председател..."` | Нормализиран текст — малки букви, без скобести пояснения |
| `word_count` | int | `94` | Брой токени (думи ≥ 3 символа) |

### Пример за валиден ред

```csv
session_date,statement_id,statement_number,mp_id,speaker,position,party,text,word_count
2026-04-01,681213,7,1433,Йордан Иванов,parl-member,Продължаваме промяната - Демократична България,"уважаеми господин председател уважаеми колеги народни представители подозирам колеги герб",94
```

## Изисквания към колоните

### `text` — най-важната колона

Трябва да е резултат от `clean_text()`:
- **Малки букви** — `text.lower()`
- **Без скобести пояснения** — `(Смях от ГЕРБ)`, `(Реплика от...)` са премахнати
- **Без излишни интервали** — свити до единичен интервал
- **Запазва пунктуацията** — Фаза 2 сама токенизира, не е нужно да е pre-tokenized
- **НЕ е токенизиран** — `text` трябва да е пълно изречение, не списък от думи

```python
# Правилно:
"уважаеми господин председател, уважаеми колеги! подозирам, че нашите колеги..."

# Грешно (pre-tokenized):
"уважаеми господин председател уважаеми колеги подозирам нашите колеги"
```

### `party` — втората критична колона

- Трябва да е **точното официално название** — Фаза 2 прави `groupby("party")`
- **Без `"Неизвестна"`** в значими изказвания — `load_speeches()` ги изфилтрира
- Проверете, че независимите депутати са `"Нечленуващи в ПГ"`, а не `"Неизвестна"`

Очаквани стойности за 51-во НС:
```
ГЕРБ-СДС
Продължаваме промяната - Демократична България
Възраждане
ДПС - Ново начало
БСП - Обединена левица
Демокрация, права и свободи
Има такъв народ
МЕЧ
Величие
Нечленуващи в ПГ
```

### `word_count` — информативна, не критична

Използва се само за филтриране на много кратки изказвания (`word_count >= 10`). Не се използва за изчисления в Фаза 2.

---

## Открити несъответствия и решения

### Несъответствие 1 — `word_count` се изчислява по различен начин

**Проблем:**

`parse_and_save.py` (Фаза 1) изчислява `word_count` с прост TOKEN_RE **без стоп-думи**:

```python
# Фаза 1 — parse_and_save.py
TOKEN_RE = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")
tokens = tokenize(speech["text"])   # ← няма стоп-думи
word_count = len(tokens)            # ← включва "ще", "това", "моля"...
```

`common.py` (Фаза 2) токенизира **с премахване на стоп-думи**:

```python
# Фаза 2 — common.py
def tokenize(text: str) -> list[str]:
    words = TOKEN_RE.findall(text.lower())
    return [w for w in words if w not in STOP_WORDS]  # ← стоп-думите са премахнати
```

**Последствие:** Изказване с `word_count=12` в CSV може да има само 4 смислени думи след анализа. Филтърът `>= 10` в `load_speeches()` няма да го отхвърли, но реалният анализ ще работи с малко думи.

**Решение — промяна в `parse_and_save.py`:**

```python
# ПРЕДИ (оригинален код):
TOKEN_RE = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")

def tokenize(text: str) -> list[str]:
    return TOKEN_RE.findall(text.lower())

# СЛЕД (синхронизиран с Фаза 2):
from common import tokenize  # ← импортираме директно от common.py
# или ако common.py не е достъпен, копираме STOP_WORDS тук

STOP_WORDS = {
    "ще", "има", "това", "които", "като", "при", "тук", "там",
    "сме", "сте", "съм", "беше", "бяха", "може", "трябва",
    "уважаеми", "уважаема", "господин", "госпожо", "колеги",
    "народни", "представители", "председател", "заповядайте",
    "моля", "благодаря", "реплика", "народния", "представител",
}

TOKEN_RE = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")

def tokenize(text: str) -> list[str]:
    words = TOKEN_RE.findall(text.lower())
    return [w for w in words if w not in STOP_WORDS]
```

**Алтернативно решение** (ако не искаш да промениш Фаза 1): промени прага в `load_speeches()` от `>= 10` на `>= 5`, за да компенсираш завишения `word_count`.

---

### Несъответствие 2 — дублирана токенизираща логика

**Проблем:** `parse_and_save.py` и `common.py` дефинират TOKEN_RE независимо.

**Решение:** `parse_and_save.py` трябва да импортира от `common.py`, или двата файла трябва да са в една директория `src/` и `parse_and_save.py` да прави:

```python
import sys
sys.path.insert(0, "src")
from common import tokenize, STOP_WORDS
```

---

### Несъответствие 3 — редът на операциите в `parse_and_save.py`

**Проблем:** Текущият код изчислява `word_count` от `speech["text"]` (суров текст) **преди** `clean_text()`:

```python
# ТЕКУЩ ред (ГРЕШЕН):
tokens = tokenize(speech["text"])    # ← суров текст, съдържа "(Смях от ГЕРБ)" и т.н.
if len(tokens) < 5:
    continue

output_rows.append({
    ...
    "text": clean_text(speech["text"]),  # ← clean се прилага след броенето
    "word_count": len(tokens),
})
```

**Решение — правилен ред:**

```python
# КОРИГИРАН ред:
clean = clean_text(speech["text"])   # ← първо почистваме
tokens = tokenize(clean)             # ← после токенизираме чистия текст
if len(tokens) < 5:
    continue

output_rows.append({
    ...
    "text": clean,                   # ← записваме чистия текст
    "word_count": len(tokens),       # ← word_count от чистия текст
})
```

## Окончателен `parse_and_save.py` (коригирана версия)

```python
# src/parse_and_save.py  — ФИНАЛНА версия, съвместима с Фаза 2
import json
import csv
import re
import sys
from pathlib import Path

# Импортираме от common.py за да избегнем дублирана логика
sys.path.insert(0, str(Path(__file__).parent))
from common import tokenize, clean_text  # ← ВАЖНО: единствен source of truth

RAW_DIR    = Path("data/raw")
PROC_DIR   = Path("data/processed")
PROC_DIR.mkdir(parents=True, exist_ok=True)


def process_all_sessions() -> None:
    output_rows = []
    party_stats: dict[str, int] = {}
    skipped = 0

    for json_file in sorted(RAW_DIR.glob("*.json")):
        session_date = json_file.stem
        with open(json_file, encoding="utf-8") as f:
            speeches = json.load(f)

        for speech in speeches:
            # 1. Почистваме текста ПЪРВО
            clean = clean_text(speech["text"])

            # 2. Токенизираме чистия текст (с STOP_WORDS от common.py)
            tokens = tokenize(clean)

            # 3. Пропускаме много кратки изказвания
            if len(tokens) < 5:
                skipped += 1
                continue

            party = speech.get("party", "Неизвестна")
            party_stats[party] = party_stats.get(party, 0) + 1

            output_rows.append({
                "session_date":     session_date,
                "statement_id":     speech["id"],
                "statement_number": speech["number"],
                "mp_id":            speech["parlMemberId"],
                "speaker":          speech["title"],
                "position":         speech["position"],
                "party":            party,
                "text":             clean,         # ← почистен, НЕ токенизиран
                "word_count":       len(tokens),   # ← след стоп-думи
            })

    # Записваме CSV
    csv_path = PROC_DIR / "speeches.csv"
    fieldnames = [
        "session_date", "statement_id", "statement_number",
        "mp_id", "speaker", "position", "party", "text", "word_count",
    ]
    with open(csv_path, "w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()
        writer.writerows(output_rows)

    print(f"Записани {len(output_rows)} изказвания в {csv_path}")
    print(f"Пропуснати (< 5 думи): {skipped}")
    print(f"\nИзказвания по партия:")
    for party, count in sorted(party_stats.items(), key=lambda x: -x[1]):
        print(f"  {party:55s}  {count:5d}")


if __name__ == "__main__":
    process_all_sessions()
```

Необходима е и следната малка промяна в `common.py` — функцията `clean_text` трябва да е там, не в `parse_and_save.py`:

```python
# src/common.py — добавяме clean_text тук
import re

TOKEN_RE  = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")

STOP_WORDS = {
    "ще", "има", "това", "които", "като", "при", "тук", "там",
    "сме", "сте", "съм", "беше", "бяха", "може", "трябва",
    "уважаеми", "уважаема", "господин", "госпожо", "колеги",
    "народни", "представители", "председател", "заповядайте",
    "моля", "благодаря", "реплика", "народния", "представител",
}


def clean_text(text: str) -> str:
    """Нормализира изказване: малки букви, без скоби, без излишни интервали."""
    text = text.lower()
    text = re.sub(r"\([^)]*\)", " ", text)   # премахва (Смях от ГЕРБ) и под.
    text = re.sub(r"\s+", " ", text).strip()
    return text


def tokenize(text: str) -> list[str]:
    """Токенизира текст: само думи ≥ 3 символа, без стоп-думи."""
    words = TOKEN_RE.findall(text.lower())
    return [w for w in words if w not in STOP_WORDS]


def make_bigrams(tokens: list[str]) -> list[str]:
    return [f"{tokens[i]}_{tokens[i+1]}" for i in range(len(tokens) - 1)]


def make_trigrams(tokens: list[str]) -> list[str]:
    return [f"{tokens[i]}_{tokens[i+1]}_{tokens[i+2]}" for i in range(len(tokens) - 2)]
```

# scripts/check_interface.py
import pandas as pd
from pathlib import Path

df = pd.read_csv("data/processed/speeches.csv", encoding="utf-8")

print("=== Проверка на интерфейса Фаза 1 → Фаза 2 ===\n")

expected_cols = {"session_date", "statement_id", "statement_number",
                 "mp_id", "speaker", "position", "party", "text", "word_count"}
missing = expected_cols - set(df.columns)

## Резюме на промените

| Файл | Промяна | Причина |
|------|---------|---------|
| `src/common.py` | Добавяме `clean_text()` тук | Единствен source of truth за нормализация |
| `src/parse_and_save.py` | Импортира `tokenize` и `clean_text` от `common.py` | Избягваме дублиране |
| `src/parse_and_save.py` | Прилагаме `clean_text()` ПРЕДИ `tokenize()` | `word_count` отразява реалния брой смислени думи |
| `src/parse_and_save.py` | Записваме `text = clean` (не суров текст) | Фаза 2 получава вече нормализиран текст |
