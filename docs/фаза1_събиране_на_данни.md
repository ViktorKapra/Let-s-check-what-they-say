## 1. Разбиране на източника на данни

### Какво е strazha.bg?

Strazha.bg е българска гражданска платформа за прозрачност, която агрегира данни от официалния сайт на Народното събрание (`parliament.bg`). Организира парламентарните стенограми по заседание, народен представител и парламентарна група (партия). Данните достигат назад до 2001 година.

За нашите цели всяко публично пленарно заседание е достъпно на предвидим URL адрес:

```
https://www.strazha.bg/sessions/{ДАТА}/steno/{НОМЕР_НА_ИЗКАЗВАНЕ}/
```

Например:
```
https://www.strazha.bg/sessions/2026-04-01/steno/6/
```

### Как са структурирани данните на страницата

Strazha.bg е изградена с **Remix.js** — React-базиран фреймуърк. Това означава, че когато сървърът рендерира страница, той вгражда всички необходими данни директно в HTML-а като JSON обект с име `window.__remixContext`. Това е ключовото откритие: **не ни трябва JavaScript изпълнение или браузърна автоматизация** — обикновена `GET` заявка връща HTML, който вече съдържа всички данни като JSON.

Ето опростен пример как изглежда вграденият JSON:

### Важна подробност за пагинацията

Всяка страница зарежда само **5 изказвания около заявения номер** (`initRange`). Едно заседание като 1 април 2026 г. съдържа **282 изказвания**. За да съберем всички изказвания от едно заседание, трябва да заявим всеки номер от 1 до 282. Метаданните за сесията (партийни членства, списък на депутатите, парламентарни групи) се **повтарят на всяка страница** — затова е достатъчно да ги парсираме само веднъж на заседание.

## 2. Правни и етични съображения

Преди да напишем и ред код за скрейпинг, е важно да изясним правния и етичен статус на тези данни.

- Всички парламентарни стенограми са **обществена информация** по българското право (ЗДОИ — Закон за достъп до обществена информация, чл. 4, ал. 1 и 3). Всеки гражданин има право на достъп.
- Strazha.bg сам заявява в своя footer: *„Всички данни представени на този сайт са обществена информация и са взети от публично достъпни източници, включително от Народното събрание на Република България."*
- Платформата е нестопанска гражданска фондация, финансирана от малки дарения. Създадена е именно с цел достъп до тези данни.
- **Добра практика:** ограничи конкурентните заявки с `asyncio.Semaphore` (препоръчително: максимум 5–10 едновременни). Това предпазва сървъра от претоварване. При изтегляне на много голям корпус добави малка пауза между партидите заявки.
- За университетски проект с разумен обем данни е добра учтивост да изпратиш кратък имейл до екипа на Стража с обяснение на целта, но не е задължително.

## 4. Имплементация стъпка по стъпка

### Стъпка 4.1 — Намиране на достъпните заседания

Преди да скрейпваме изказвания, трябва да знаем кои заседания съществуват и датите им. Навигираме до:

```
https://www.strazha.bg/sessions/
```

Същият `window.__remixContext` модел важи. Парсираме списъка от дати (slugове) на заседанията. За проекта ще използваме **три размера на корпуса**:

| Корпус | Заседания | Приблизителни изказвания |
|--------|-----------|--------------------------|
| Малък  | 1 заседание | ~280 |
| Среден | ~10 заседания (1 месец) | ~2 500 |
| Голям  | ~120 заседания (1 година) | ~35 000 |

### Стъпка 4.2 — Серийно теглене (`scraper_serial.py`)

Това е **базовото решение (baseline)** — последователен цикъл, който събира всички изказвания от едно заседание, по една HTTP заявка наведнъж. Изпълнява две роли: (1) работещата референтна имплементация, която по-късно ще паралелизираме, и (2) нейното времетраене е знаменателят в нашата формула за speedup.

```python
import requests
import re
import json
import time
from pathlib import Path

# --- Конфигурация ---
BASE_URL = "https://www.strazha.bg"
HEADERS = {
    "User-Agent": (
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) "
        "AppleWebKit/537.36 (KHTML, like Gecko) "
        "Chrome/124.0.0.0 Safari/537.36"
    ),
    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
    "Accept-Language": "bg-BG,bg;q=0.9",
    "Referer": "https://www.strazha.bg/",
}
# Regex за извличане на вградения JSON от HTML-а
REMIX_RE = re.compile(r"window\.__remixContext\s*=\s*(\{.*?\});\s*</script>", re.DOTALL)


def fetch_html(url: str) -> str:
    """Извлича URL и връща суровия HTML. Хвърля изключение при HTTP грешки."""
    response = requests.get(url, headers=HEADERS, timeout=15)
    response.raise_for_status()
    return response.text


def parse_remix_context(html: str) -> dict:
    """Извлича и парсира JSON-а от window.__remixContext от суровия HTML."""
    match = REMIX_RE.search(html)
    if not match:
        raise ValueError("window.__remixContext не е намерен на страницата")
    return json.loads(match.group(1))


def extract_session_metadata(context: dict) -> dict:
    """
    Извлича партийните групи и членствата на депутатите от контекста на заседанието.
    Тези данни са еднакви на всяка страница на заседанието, затова ги парсираме само веднъж.

    Връща речник с parlMemberId -> name на партия.
    """
    loader = context["state"]["loaderData"]
    session_data = loader["routes/_front.sessions.$sessionSlug"]

    # Справочник: parlGroupId -> name на партия
    party_by_id = {
        pg["id"]: pg["name"]
        for pg in session_data.get("parlGroups", [])
    }

    # Справочник: parlMemberId -> parlGroupId
    group_by_member = {
        m["parlMemberId"]: m["parlGroupId"]
        for m in session_data.get("pgMemberships", [])
    }

    # Комбинираме: parlMemberId -> name на партия
    party_by_member = {
        member_id: party_by_id.get(group_id, "Неизвестна")
        for member_id, group_id in group_by_member.items()
    }

    return {
        "party_by_member": party_by_member,
        "party_by_id": party_by_id,
        "statement_count": int(session_data["parlSession"].get("statementCount", 0)),
        "session_date": session_data["parlSession"]["slug"],
    }


def extract_statements(context: dict) -> list[dict]:
    """
    Извлича списъка с изказвания от контекста на една страница.
    Връща списък от речници с ключове: id, number, parlMemberId, title, text.
    """
    loader = context["state"]["loaderData"]
    steno_data = loader["routes/_front.sessions.$sessionSlug.steno"]
    raw_statements = steno_data.get("sessionStatements", [])

    statements = []
    for s in raw_statements:
        statements.append({
            "id": s["id"],
            "number": s["number"],
            "parlMemberId": s["parlMemberId"],
            "title": s["title"],
            "position": s["position"],        # "house-speaker" или "parl-member"
            "text": " ".join(s["paragraphs"]), # обединяваме всички параграфи в един текст
        })
    return statements


def scrape_session_serial(date: str, output_dir: Path) -> list[dict]:
    """
    Скрейпва ВСИЧКИ изказвания от едно заседание, идентифицирано по дата.
    Извлича по едно изказване наведнъж (серийно).

    Аргументи:
        date: дата на заседанието като стринг, напр. "2026-04-01"
        output_dir: директория за запис на суровия JSON

    Връща:
        списък от речници с изказвания, обогатени с партиен etикет
    """
    output_dir.mkdir(parents=True, exist_ok=True)

    # --- Стъпка 1: Извличаме метаданните на заседанието от изказване №1 ---
    first_url = f"{BASE_URL}/sessions/{date}/steno/1/"
    print(f"[серийно] Извличам метаданни от {first_url}")
    html = fetch_html(first_url)
    context = parse_remix_context(html)
    meta = extract_session_metadata(context)

    total = meta["statement_count"]
    party_by_member = meta["party_by_member"]
    print(f"[серийно] Заседание {date}: {total} изказвания, "
          f"{len(meta['party_by_id'])} партии")

    # --- Стъпка 2: Събираме всички изказвания като итерираме по номерата ---
    all_statements: dict[int, dict] = {}

    for stmt_num in range(1, total + 1):
        url = f"{BASE_URL}/sessions/{date}/steno/{stmt_num}/"
        try:
            html = fetch_html(url)
            ctx = parse_remix_context(html)
            for stmt in extract_statements(ctx):
                # Избягваме дубликати — страницата зарежда 5 изказвания около заявеното
                if stmt["number"] not in all_statements:
                    all_statements[stmt["number"]] = stmt
        except Exception as e:
            print(f"[серийно] Предупреждение: неуспешно изказване {stmt_num}: {e}")

        # Учтива пауза за да не претоварваме сървъра
        time.sleep(0.1)

    # --- Стъпка 3: Обогатяваме с партийни етикети ---
    result = []
    for stmt in sorted(all_statements.values(), key=lambda s: s["number"]):
        member_id = stmt["parlMemberId"]
        stmt["party"] = party_by_member.get(member_id, "Неизвестна")
        result.append(stmt)

    # --- Стъпка 4: Записваме суровите данни ---
    out_path = output_dir / f"{date}.json"
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump(result, f, ensure_ascii=False, indent=2)
    print(f"[серийно] Записани {len(result)} изказвания в {out_path}")

    return result


# --- Входна точка ---
if __name__ == "__main__":
    import time as _time

    RAW_DIR = Path("data/raw")
    SESSION_DATE = "2026-04-01"

    start = _time.perf_counter()
    speeches = scrape_session_serial(SESSION_DATE, RAW_DIR)
    elapsed = _time.perf_counter() - start

    print(f"\n[серийно] Готово. {len(speeches)} изказвания за {elapsed:.2f} сек.")
```

**Какво да наблюдаваме при стартиране:**
- Общото времетраене за едно заседание (~280 изказвания) вероятно ще е 30–90 секунди в зависимост от мрежовата латентност и паузата от 0.1 сек.
- Това време е нашето **T_серийно** — знаменателят за пресмятане на speedup в Фаза 3.
- Проверяваме, че полето `party` е правилно попълнено за всяко изказване.

### Стъпка 4.3 — Асинхронно теглене (`scraper_async.py`)

Асинхронната версия замества последователния цикъл `requests` + `time.sleep` с `aiohttp` и `asyncio`. Вместо да чакаме всеки HTTP отговор преди да изпратим следващата заявка, изпращаме много заявки едновременно и ги обработваме при получаване.

Това е класическа **I/O-bound** задача — процесорът почти изцяло стои в очакване на сървърните отговори. Именно за такива проблеми е създаден `asyncio`.

```python
import asyncio
import aiohttp
import re
import json
import time
from pathlib import Path

BASE_URL = "https://www.strazha.bg"
HEADERS = {
    "User-Agent": (
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) "
        "AppleWebKit/537.36 (KHTML, like Gecko) "
        "Chrome/124.0.0.0 Safari/537.36"
    ),
    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8",
    "Accept-Language": "bg-BG,bg;q=0.9",
    "Referer": "https://www.strazha.bg/",
}
REMIX_RE = re.compile(r"window\.__remixContext\s*=\s*(\{.*?\});\s*</script>", re.DOTALL)

# Семафорът ограничава броя на едновременните заявки.
# 5–10 е учтива стойност спрямо сървъра.
MAX_CONCURRENT = 10


async def fetch_html_async(
    session: aiohttp.ClientSession,
    url: str,
    semaphore: asyncio.Semaphore,
) -> str:
    """Асинхронна версия на fetch_html. Използва споделения семафор за ограничаване."""
    async with semaphore:
        async with session.get(url, headers=HEADERS, timeout=aiohttp.ClientTimeout(total=15)) as resp:
            resp.raise_for_status()
            return await resp.text()


def parse_remix_context(html: str) -> dict:
    match = REMIX_RE.search(html)
    if not match:
        raise ValueError("window.__remixContext не е намерен на страницата")
    return json.loads(match.group(1))


def extract_statements(context: dict) -> list[dict]:
    loader = context["state"]["loaderData"]
    steno_data = loader["routes/_front.sessions.$sessionSlug.steno"]
    return [
        {
            "id": s["id"],
            "number": s["number"],
            "parlMemberId": s["parlMemberId"],
            "title": s["title"],
            "position": s["position"],
            "text": " ".join(s["paragraphs"]),
        }
        for s in steno_data.get("sessionStatements", [])
    ]


def extract_session_metadata(context: dict) -> dict:
    loader = context["state"]["loaderData"]
    session_data = loader["routes/_front.sessions.$sessionSlug"]
    party_by_id = {pg["id"]: pg["name"] for pg in session_data.get("parlGroups", [])}
    group_by_member = {m["parlMemberId"]: m["parlGroupId"] for m in session_data.get("pgMemberships", [])}
    return {
        "party_by_member": {mid: party_by_id.get(gid, "Неизвестна") for mid, gid in group_by_member.items()},
        "party_by_id": party_by_id,
        "statement_count": int(session_data["parlSession"].get("statementCount", 0)),
        "session_date": session_data["parlSession"]["slug"],
    }


async def scrape_session_async(
    date: str,
    output_dir: Path,
    max_concurrent: int = MAX_CONCURRENT,
) -> list[dict]:
    """
    Скрейпва ВСИЧКИ изказвания от заседание, като изпраща до `max_concurrent`
    HTTP заявки едновременно.
    """
    output_dir.mkdir(parents=True, exist_ok=True)
    semaphore = asyncio.Semaphore(max_concurrent)

    async with aiohttp.ClientSession() as session:
        # --- Стъпка 1: Метаданни от първата страница ---
        first_url = f"{BASE_URL}/sessions/{date}/steno/1/"
        print(f"[asyncio] Извличам метаданни от {first_url}")
        first_html = await fetch_html_async(session, first_url, semaphore)
        context = parse_remix_context(first_html)
        meta = extract_session_metadata(context)

        total = meta["statement_count"]
        party_by_member = meta["party_by_member"]
        print(f"[asyncio] Заседание {date}: {total} изказвания, конкурентност={max_concurrent}")

        # --- Стъпка 2: Изграждаме всички URL-и ---
        urls = [
            (stmt_num, f"{BASE_URL}/sessions/{date}/steno/{stmt_num}/")
            for stmt_num in range(1, total + 1)
        ]

        # --- Стъпка 3: Извличаме всички страници конкурентно ---
        async def fetch_one(stmt_num: int, url: str) -> tuple[int, list[dict]]:
            try:
                html = await fetch_html_async(session, url, semaphore)
                ctx = parse_remix_context(html)
                return stmt_num, extract_statements(ctx)
            except Exception as e:
                print(f"[asyncio] Предупреждение: неуспешно изказване {stmt_num}: {e}")
                return stmt_num, []

        tasks = [fetch_one(num, url) for num, url in urls]
        results = await asyncio.gather(*tasks)

    # --- Стъпка 4: Премахване на дубликати и сортиране ---
    all_statements: dict[int, dict] = {}
    for _, stmts in results:
        for stmt in stmts:
            if stmt["number"] not in all_statements:
                all_statements[stmt["number"]] = stmt

    # --- Стъпка 5: Обогатяваме с партийни етикети и записваме ---
    final = []
    for stmt in sorted(all_statements.values(), key=lambda s: s["number"]):
        stmt["party"] = party_by_member.get(stmt["parlMemberId"], "Неизвестна")
        final.append(stmt)

    out_path = output_dir / f"{date}.json"
    with open(out_path, "w", encoding="utf-8") as f:
        json.dump(final, f, ensure_ascii=False, indent=2)
    print(f"[asyncio] Записани {len(final)} изказвания в {out_path}")

    return final


# --- Входна точка ---
if __name__ == "__main__":
    RAW_DIR = Path("data/raw")
    SESSION_DATE = "2026-04-01"

    start = time.perf_counter()
    speeches = asyncio.run(scrape_session_async(SESSION_DATE, RAW_DIR))
    elapsed = time.perf_counter() - start

    print(f"\n[asyncio] Готово. {len(speeches)} изказвания за {elapsed:.2f} сек.")
```

**Какво да наблюдаваме:**
- При `max_concurrent=10` асинхронният скрейпър трябва да е приблизително 5–10 пъти по-бърз от серийния за реално заседание.
- Опитай различни стойности: 1, 5, 10, 20. Начертай графика на времето спрямо конкурентността — това само по себе си е интересна графика за доклада.
- При много висока конкурентност (50+) сървърът може да започне да ограничава заявките или да връща грешки. Семафорът предотвратява това.

### Стъпка 4.4 — Парсиране и запис (`parse_and_save.py`)

След скрейпинга конвертираме суровите JSON файлове в чист CSV, който кодът за анализ във Фаза 2 ще използва. Всеки ред е едно изказване с пълния му текст и партиен етикет.

```python
import json
import csv
import re
from pathlib import Path

RAW_DIR = Path("data/raw")
PROCESSED_DIR = Path("data/processed")
PROCESSED_DIR.mkdir(parents=True, exist_ok=True)

# Прост токенизатор за български: запазва думи от 3+ символа
TOKEN_RE = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")


def clean_text(text: str) -> str:
    """
    Нормализира изказване: малки букви, премахва пунктуация и шум.
    Запазва само български и латински символи.
    """
    text = text.lower()
    # Премахване на сценични указания в скоби като (Смях от ГЕРБ)
    text = re.sub(r"\([^)]*\)", " ", text)
    # Свиване на празни места
    text = re.sub(r"\s+", " ", text).strip()
    return text


def tokenize(text: str) -> list[str]:
    return TOKEN_RE.findall(text.lower())


def process_all_sessions():
    output_rows = []
    party_stats: dict[str, int] = {}

    for json_file in sorted(RAW_DIR.glob("*.json")):
        session_date = json_file.stem
        with open(json_file, encoding="utf-8") as f:
            speeches = json.load(f)

        for speech in speeches:
            # Пропускаме процедурни изказвания с по-малко от 5 думи
            tokens = tokenize(speech["text"])
            if len(tokens) < 5:
                continue

            party = speech.get("party", "Неизвестна")
            party_stats[party] = party_stats.get(party, 0) + 1

            output_rows.append({
                "session_date": session_date,
                "statement_id": speech["id"],
                "statement_number": speech["number"],
                "mp_id": speech["parlMemberId"],
                "speaker": speech["title"],
                "position": speech["position"],
                "party": party,
                "text": clean_text(speech["text"]),
                "word_count": len(tokens),
            })

    # Записваме CSV
    csv_path = PROCESSED_DIR / "speeches.csv"
    fieldnames = ["session_date", "statement_id", "statement_number",
                  "mp_id", "speaker", "position", "party", "text", "word_count"]

    with open(csv_path, "w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()
        writer.writerows(output_rows)

    print(f"Записани {len(output_rows)} изказвания в {csv_path}")
    print("\nИзказвания по партия:")
    for party, count in sorted(party_stats.items(), key=lambda x: -x[1]):
        print(f"  {party:55s}  {count:5d}")


if __name__ == "__main__":
    process_all_sessions()
```

## 8. Чести проблеми и решения

| Проблем | Причина | Решение |
|---------|---------|---------|
| `window.__remixContext не е намерен` | Страницата върна грешка или пренасочване | Провери формата на URL-а; отпечатай `response.status_code` |
| Липсващи `pgMemberships` | Структурата на метаданните е променена | Отпечатай ключовете от най-високо ниво на `session_data` за дебъгване |
| Много "Неизвестна" партии | Депутатът е влязъл по средата на заседанието или е независим | Очаквано за ~5% от изказванията; документирай го |
| `asyncio.TimeoutError` | Сървърът е бавен или има твърде много конкурентни заявки | Намали `MAX_CONCURRENT` или увеличи timeout-а |
| Дублирани изказвания | Причинени от припокриващи се прозорци на `initRange` | Стъпката за дедупликация чрез речника `all_statements` се справя с това |
| Грешки с Unicode в CSV | Кодирането по подразбиране на Windows | Винаги отваряй файлове с `encoding="utf-8"` |
