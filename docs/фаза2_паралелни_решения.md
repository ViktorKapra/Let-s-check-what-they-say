# Фаза 2 — Имплементация на серийно и паралелни решения

**Проект:** Паралелен анализ на политическия език в българския парламент  
**Входни данни:** `data/processed/speeches.csv` (изход от Фаза 1)  
**Очаквано време:** 6–8 часа  
**Цел:** Реализация на четири версии на един и същ анализ — серийна (baseline) и три паралелни — с ясна обосновка защо всеки подход е подходящ за конкретната стъпка от пайплайна.

---

## Преглед на пайплайна

Анализът се разпада на три независими стъпки. Всяка стъпка се паралелизира по различен начин:

```
speeches.csv
    │
    ├── Стъпка A: Токенизация и честотен анализ (Word Count, N-gram)
    │           → CPU-bound, независима по документ
    │           → Паралелизира се с multiprocessing.Pool
    │
    ├── Стъпка Б: TF-IDF матрица и косинусово сходство
    │           → Data-parallel, масивни операции
    │           → Паралелизира се с NumPy / scipy.sparse
    │
    └── Стъпка В: Агрегиране и сравнение по партии (голям корпус)
                → Distributed, по-голям от RAM
                → Паралелизира се с Dask
```

Всяка стъпка има серийна версия (baseline) и паралелна версия. Кодът е организиран така, че двете версии приемат **еднакъв вход и произвеждат еднакъв изход** — единствената разлика е времетраенето.

## Подготовка — зареждане на данните

Преди да започнем с имплементацията, дефинираме общите структури, които всички файлове ще използват.

```python
# src/common.py
import pandas as pd
import re
from collections import Counter
from pathlib import Path

# Прост токенизатор за български: само думи от 3+ символа
TOKEN_RE = re.compile(r"\b[а-яА-Яa-zA-Z]{3,}\b")

# Стоп-думи за български — думи без аналитична стойност
STOP_WORDS = {
    "ще", "има", "това", "които", "като", "при", "тук", "там",
    "сме", "сте", "съм", "беше", "бяха", "може", "трябва",
    "уважаеми", "уважаема", "господин", "госпожо", "колеги",
    "народни", "представители", "председател", "заповядайте",
    "моля", "благодаря", "реплика", "народния", "представител",
}


def load_speeches(csv_path: Path) -> pd.DataFrame:
    """Зарежда обработените изказвания от CSV файла."""
    df = pd.read_csv(csv_path, encoding="utf-8")
    df = df[df["party"] != "Неизвестна"].copy()
    df = df[df["word_count"] >= 10].copy()
    print(f"Заредени {len(df)} изказвания от {df['party'].nunique()} партии")
    return df


def tokenize(text: str) -> list[str]:
    """Токенизира текст: малки букви, само думи, без стоп-думи."""
    words = TOKEN_RE.findall(text.lower())
    return [w for w in words if w not in STOP_WORDS]


def make_bigrams(tokens: list[str]) -> list[str]:
    """Генерира биграми от списък с токени."""
    return [f"{tokens[i]}_{tokens[i+1]}" for i in range(len(tokens) - 1)]


def make_trigrams(tokens: list[str]) -> list[str]:
    """Генерира триграми от списък с токени."""
    return [f"{tokens[i]}_{tokens[i+1]}_{tokens[i+2]}" for i in range(len(tokens) - 2)]
```

## 2.0 Серийно решение — baseline (`analysis_serial.py`)

Серийното решение е **задължително** и служи за три цели:
1. Доказва коректността на алгоритъма
2. Служи като отправна точка (T_serial) за пресмятане на speedup
3. Трябва да е разбираемо и добре коментирано — то обяснява **какво** правим, паралелните версии обясняват **как го ускоряваме**

```python
# src/analysis_serial.py
"""
Серийно (baseline) решение за анализ на политическия език.

Алгоритъм:
1. За всяко изказване: токенизация, word count, n-gram
2. Агрегиране по партия
3. TF-IDF изчисление
4. Извличане на топ думи и n-грами по партия
"""

import time
import json
from collections import Counter, defaultdict
from pathlib import Path
import pandas as pd
from common import load_speeches, tokenize, make_bigrams, make_trigrams

# ── Стъпка A: Анализ на едно изказване ──────────────────────────────────────

def analyze_one_speech(speech: dict) -> dict:
    """
    Анализира едно изказване и връща честотни данни.

    Входен формат:
        {"text": "...", "party": "ГЕРБ-СДС", "statement_id": 12345, ...}

    Изходен формат:
        {"party": "...", "unigrams": Counter, "bigrams": Counter, "trigrams": Counter}
    """
    tokens = tokenize(speech["text"])
    bigrams = make_bigrams(tokens)
    trigrams = make_trigrams(tokens)

    return {
        "party": speech["party"],
        "unigrams": Counter(tokens),
        "bigrams": Counter(bigrams),
        "trigrams": Counter(trigrams),
        "total_words": len(tokens),
    }


# ── Стъпка Б: Агрегиране по партия ──────────────────────────────────────────

def aggregate_by_party(results: list[dict]) -> dict:
    """
    Обединява резултатите от всички изказвания по партия.

    Входен формат: списък от резултати от analyze_one_speech()
    Изходен формат: {party: {"unigrams": Counter, "bigrams": Counter, ...}}
    """
    by_party = defaultdict(lambda: {
        "unigrams": Counter(),
        "bigrams": Counter(),
        "trigrams": Counter(),
        "total_words": 0,
        "speech_count": 0,
    })

    for r in results:
        party = r["party"]
        by_party[party]["unigrams"].update(r["unigrams"])
        by_party[party]["bigrams"].update(r["bigrams"])
        by_party[party]["trigrams"].update(r["trigrams"])
        by_party[party]["total_words"] += r["total_words"]
        by_party[party]["speech_count"] += 1

    return dict(by_party)


# ── Стъпка В: TF-IDF изчисление ─────────────────────────────────────────────

def compute_tfidf(by_party: dict, top_n: int = 20) -> dict:
    """
    Изчислява TF-IDF за всяка дума във всяка партия.

    TF  = брой срещания на думата в партията / общ брой думи в партията
    IDF = log(брой партии / брой партии в които се среща думата)
    TF-IDF = TF * IDF

    Думи с висок TF-IDF са характерни САМО за тази партия, не за всички.
    """
    import math

    parties = list(by_party.keys())
    n_parties = len(parties)

    # Пресмятаме в колко партии се среща всяка дума
    doc_freq: Counter = Counter()
    for party_data in by_party.values():
        doc_freq.update(set(by_party[party_data["unigrams"] if False else "unigrams"] if False else party_data["unigrams"].keys()))

    # За всяка партия изчисляваме TF-IDF
    tfidf_results = {}
    for party, data in by_party.items():
        total = data["total_words"]
        if total == 0:
            continue

        scores = {}
        for word, count in data["unigrams"].items():
            tf = count / total
            idf = math.log(n_parties / (doc_freq[word] + 1))
            scores[word] = tf * idf

        # Топ N думи по TF-IDF
        tfidf_results[party] = sorted(scores.items(), key=lambda x: -x[1])[:top_n]

    return tfidf_results


# ── Основна функция ──────────────────────────────────────────────────────────

def run_serial(df: pd.DataFrame) -> dict:
    """
    Изпълнява пълния серийен анализ.

    Сложност: O(N * L) — N изказвания, L средна дължина на изказване
    """
    speeches = df.to_dict("records")

    # A: Анализ на всяко изказване — серийно
    print(f"  Анализирам {len(speeches)} изказвания серийно...")
    speech_results = [analyze_one_speech(s) for s in speeches]

    # Б: Агрегиране по партия
    print("  Агрегирам по партия...")
    by_party = aggregate_by_party(speech_results)

    # В: TF-IDF
    print("  Изчислявам TF-IDF...")
    tfidf = compute_tfidf(by_party)

    return {"by_party": by_party, "tfidf": tfidf}


# ── Входна точка ─────────────────────────────────────────────────────────────

if __name__ == "__main__":
    DATA_PATH = Path("data/processed/speeches.csv")
    df = load_speeches(DATA_PATH)

    start = time.perf_counter()
    results = run_serial(df)
    elapsed = time.perf_counter() - start

    print(f"\nСерийно: {elapsed:.3f} сек.")
    print("\nТоп 10 думи по TF-IDF за всяка партия:")
    for party, words in results["tfidf"].items():
        top5 = ", ".join(w for w, _ in words[:5])
        print(f"  {party[:40]:40s}: {top5}")
```

## 2.1 CPU паралелизъм — `multiprocessing.Pool` (`analysis_mp.py`)

### Защо `multiprocessing` тук?

Токенизацията и честотното броене (Стъпка A) са **CPU-bound** — процесорът активно работи за всяко изказване. Python-овият GIL (Global Interpreter Lock) не позволява истински паралелен CPU код с нишки (`threading`). Решението е `multiprocessing` — всеки процес има собствен интерпретатор и собствен GIL.

Освен това Стъпка A е **embarrassingly parallel** — изказванията са напълно независими едно от друго. Нямаме нужда от синхронизация, споделена памет или комуникация. `Pool.map()` разпределя изказванията автоматично между наличните ядра.

### Ограничения

- **Overhead при сериализация:** всяко изказване трябва да се сериализира (pickle) за да се изпрати до работния процес. При много малки задачи това може да е по-бавно от серийното решение.
- **Агрегирането (Стъпка Б) е серийно:** сливането на Counter-ите изисква синхронизация и не се паралелизира директно. Това е **Amdahl's Law** в действие.

```python
# src/analysis_mp.py
"""
Паралелно решение с multiprocessing.Pool.

Паралелизира Стъпка A (анализ на всяко изказване).
Стъпките Б и В остават серийни — агрегирането изисква синхронизация.

Защо работи:
  - Всяко изказване е независимо → embarrassingly parallel
  - Заобикаля GIL чрез отделни процеси
  - Pool.map() автоматично разпределя задачите по ядра

Ограничения:
  - Overhead при pickle сериализация на всяко изказване
  - При малък корпус серийното може да е по-бързо
"""

import time
import multiprocessing as mp
from multiprocessing import Pool
from pathlib import Path
import pandas as pd
from common import load_speeches, tokenize, make_bigrams, make_trigrams
from collections import Counter, defaultdict
from analysis_serial import aggregate_by_party, compute_tfidf


# ВАЖНО: analyze_one_speech трябва да е на модулно ниво (не вложена функция)
# Pool сериализира функцията с pickle — вложените функции не могат да се pickle-ват
def analyze_one_speech(speech: dict) -> dict:
    """
    Анализира едно изказване. Идентична с серийната версия —
    единствената разлика е че тук се изпълнява в отделен процес.
    """
    tokens = tokenize(speech["text"])
    return {
        "party": speech["party"],
        "unigrams": Counter(tokens),
        "bigrams": Counter(make_bigrams(tokens)),
        "trigrams": Counter(make_trigrams(tokens)),
        "total_words": len(tokens),
    }


def run_parallel_mp(df: pd.DataFrame, n_processes: int = None) -> dict:
    """
    Изпълнява анализа с multiprocessing.Pool.

    Args:
        df: DataFrame с изказванията
        n_processes: брой процеси. None = брой физически ядра
    """
    if n_processes is None:
        n_processes = mp.cpu_count()

    speeches = df.to_dict("records")
    print(f"  Стартирам Pool с {n_processes} процеса за {len(speeches)} изказвания...")

    # Pool.map() — блокиращо, чака всички задачи да завършат
    # chunksize разпределя задачите на по-големи пакети за по-малко overhead
    chunksize = max(1, len(speeches) // (n_processes * 4))

    with Pool(processes=n_processes) as pool:
        speech_results = pool.map(analyze_one_speech, speeches, chunksize=chunksize)

    # Агрегирането е серийно — изисква синхронизация
    print("  Агрегирам по партия (серийно)...")
    by_party = aggregate_by_party(speech_results)

    print("  Изчислявам TF-IDF (серийно)...")
    tfidf = compute_tfidf(by_party)

    return {"by_party": by_party, "tfidf": tfidf}


def benchmark_mp(df: pd.DataFrame) -> dict:
    """
    Сравнява Pool.map() при различен брой процеси.
    Връща речник с времената за по-нататъшен анализ.
    """
    max_cores = mp.cpu_count()
    process_counts = [1, 2, 4, max_cores] if max_cores >= 4 else list(range(1, max_cores + 1))

    timings = {}
    for n in process_counts:
        start = time.perf_counter()
        run_parallel_mp(df, n_processes=n)
        elapsed = time.perf_counter() - start
        timings[n] = elapsed
        print(f"  {n} процеса: {elapsed:.3f} сек.")

    return timings


if __name__ == "__main__":
    DATA_PATH = Path("data/processed/speeches.csv")
    df = load_speeches(DATA_PATH)

    # Серийно за baseline
    from analysis_serial import run_serial
    start = time.perf_counter()
    run_serial(df)
    t_serial = time.perf_counter() - start
    print(f"Серийно: {t_serial:.3f} сек.\n")

    # Паралелно с Pool
    print("Паралелно с Pool.map():")
    timings = benchmark_mp(df)

    print("\nSpeedup спрямо серийното:")
    for n_proc, t in timings.items():
        speedup = t_serial / t
        efficiency = speedup / n_proc
        print(f"  {n_proc:2d} процеса: speedup={speedup:.2f}x, efficiency={efficiency:.2f}")
```

## 2.2 Векторизация — NumPy / scipy.sparse (`analysis_vectorized.py`)

### Защо векторизация тук?

TF-IDF (Стъпка Б) се свежда до матрични операции: имаме **N изказвания × V думи** матрица. Python цикли върху тази матрица са изключително бавни. NumPy и scipy изпълняват тези операции в оптимизиран C/Fortran код с SIMD инструкции — матричното умножение се ускорява многократно без нито един допълнителен процес или нишка.

За разлика от `multiprocessing`, тук **паралелизмът е в данните, не в задачите** — работим с цели масиви наведнъж вместо с отделни елементи.

```python
# src/analysis_vectorized.py
"""
Векторизирано решение с NumPy и scipy.sparse.

Паралелизира Стъпка Б (TF-IDF и косинусово сходство) чрез data parallelism.

Защо работи:
  - sklearn и scipy изпълняват матричните операции в оптимизиран C/Fortran
  - SIMD инструкции обработват множество елементи едновременно
  - Sparse матриците пестят памет при голям речник

Сравнение с ръчното TF-IDF от серийното решение:
  - Серийно: Python цикъл, O(N * V) операции в Python
  - Vectorized: numpy масиви, O(N * V) операции в C
"""

import time
import numpy as np
from scipy.sparse import csr_matrix
from sklearn.feature_extraction.text import TfidfVectorizer, CountVectorizer
from sklearn.metrics.pairwise import cosine_similarity
import pandas as pd
from pathlib import Path
from common import load_speeches, STOP_WORDS


def build_party_corpus(df: pd.DataFrame) -> tuple[list[str], list[str]]:
    """
    Обединява всички изказвания на всяка партия в един документ.
    Връща (corpus, party_names).
    """
    party_texts = df.groupby("party")["text"].apply(" ".join)
    return party_texts.tolist(), party_texts.index.tolist()


def run_tfidf_vectorized(df: pd.DataFrame, top_n: int = 20) -> dict:
    """
    Изчислява TF-IDF с sklearn.TfidfVectorizer.

    Връща:
        - tfidf матрица (партии × думи)
        - топ N думи за всяка партия
        - матрица на косинусовото сходство между партиите
    """
    corpus, parties = build_party_corpus(df)

    # TfidfVectorizer вътрешно:
    # 1. Токенизира
    # 2. Изгражда речник
    # 3. Изчислява TF-IDF матрица като scipy sparse матрица
    vectorizer = TfidfVectorizer(
        analyzer="word",
        token_pattern=r"\b[а-яА-Яa-zA-Z]{3,}\b",
        stop_words=list(STOP_WORDS),
        max_features=10_000,   # максимален речник
        min_df=2,              # думата трябва да е в поне 2 партии
        sublinear_tf=True,     # log(1 + tf) вместо сурово tf
    )

    print(f"  Изграждам TF-IDF матрица за {len(parties)} партии...")
    tfidf_matrix = vectorizer.fit_transform(corpus)
    feature_names = vectorizer.get_feature_names_out()

    print(f"  Матрица: {tfidf_matrix.shape[0]} партии × {tfidf_matrix.shape[1]} думи")
    print(f"  Sparse матрица: {tfidf_matrix.nnz} ненулеви елемента "
          f"({100 * tfidf_matrix.nnz / (tfidf_matrix.shape[0] * tfidf_matrix.shape[1]):.2f}% плътност)")

    # Топ N думи за всяка партия
    top_words = {}
    for i, party in enumerate(parties):
        scores = tfidf_matrix[i].toarray().flatten()
        top_idx = np.argsort(scores)[::-1][:top_n]
        top_words[party] = [(feature_names[j], scores[j]) for j in top_idx]

    # Косинусово сходство между партиите
    # cosine_similarity(A, B) = (A · B) / (||A|| * ||B||)
    # Стойност 1.0 = идентичен език, 0.0 = напълно различен
    print("  Изчислявам косинусово сходство между партиите...")
    similarity_matrix = cosine_similarity(tfidf_matrix)

    return {
        "tfidf_matrix": tfidf_matrix,
        "feature_names": feature_names,
        "parties": parties,
        "top_words": top_words,
        "similarity_matrix": similarity_matrix,
    }


def run_ngram_vectorized(df: pd.DataFrame, ngram_range: tuple = (2, 3)) -> dict:
    """
    Изчислява честоти на n-грами с CountVectorizer.

    ngram_range=(2,3) означава биграми И триграми едновременно.
    """
    corpus, parties = build_party_corpus(df)

    vectorizer = CountVectorizer(
        analyzer="word",
        token_pattern=r"\b[а-яА-Яa-zA-Z]{3,}\b",
        ngram_range=ngram_range,
        max_features=5_000,
        min_df=2,
    )

    print(f"  Изграждам {ngram_range}-gram матрица...")
    ngram_matrix = vectorizer.fit_transform(corpus)
    feature_names = vectorizer.get_feature_names_out()

    top_ngrams = {}
    for i, party in enumerate(parties):
        counts = ngram_matrix[i].toarray().flatten()
        top_idx = np.argsort(counts)[::-1][:20]
        top_ngrams[party] = [(feature_names[j], int(counts[j])) for j in top_idx]

    return {"top_ngrams": top_ngrams, "parties": parties}


if __name__ == "__main__":
    DATA_PATH = Path("data/processed/speeches.csv")
    df = load_speeches(DATA_PATH)

    start = time.perf_counter()
    results = run_tfidf_vectorized(df)
    elapsed = time.perf_counter() - start

    print(f"\nВекторизирано: {elapsed:.3f} сек.")

    print("\nКосинусово сходство между партиите:")
    parties = results["parties"]
    sim = results["similarity_matrix"]
    for i, p1 in enumerate(parties):
        for j, p2 in enumerate(parties):
            if i < j:
                print(f"  {p1[:25]:25s} ↔ {p2[:25]:25s}: {sim[i,j]:.3f}")

    print("\nТоп 5 думи по TF-IDF за всяка партия:")
    for party, words in results["top_words"].items():
        top5 = ", ".join(w for w, _ in words[:5])
        print(f"  {party[:40]:40s}: {top5}")
```

## 2.3 Distributed — Dask (`analysis_dask.py`)

### Защо Dask тук?

При голям корпус (35 000+ изказвания) операциите могат да надхвърлят наличната RAM памет. Dask работи **lazy** — изгражда граф на изчисленията и ги изпълнява на части, зареждайки само необходимото в паметта. Това е качествено различно от `multiprocessing` (повече ядра на един компютър) — Dask може да разпредели работата и на множество машини.

За нашия проект Dask демонстрира **distributed подхода** дори на един компютър, като разпределя данните между партициите автоматично.

```python
# src/analysis_dask.py
"""
Distributed решение с Dask.

Подходящо при голям корпус (> RAM) или при нужда от скалиране на много машини.

Защо е различно от multiprocessing:
  - multiprocessing: повече ядра на ЕДИН компютър
  - Dask: lazy evaluation + разделяне на данните на партиции
         → може да работи и на клъстер от машини
         → зарежда само необходимата памет наведнъж

Кога Dask НЕ помага:
  - При малък корпус (< 10 000 реда) overhead-ът на Dask е по-голям от ускорението
  - При прости операции без нужда от lazy evaluation
"""

import time
from pathlib import Path
import pandas as pd
import dask.dataframe as dd
import dask.bag as db
from dask.diagnostics import ProgressBar
from common import load_speeches, tokenize, make_bigrams
from collections import Counter


def run_wordcount_dask(csv_path: Path, n_partitions: int = 4) -> dict:
    """
    Word count по партия с Dask DataFrame.

    Dask разделя CSV файла на n_partitions части и ги обработва паралелно.
    """
    print(f"  Зареждам {csv_path} с Dask ({n_partitions} партиции)...")

    # Dask чете CSV lazy — не зарежда всичко в RAM
    ddf = dd.read_csv(csv_path, encoding="utf-8")
    ddf = ddf[ddf["party"] != "Неизвестна"]
    ddf = ddf.repartition(npartitions=n_partitions)

    # Броим думите по партия — Dask изпълнява groupby паралелно
    print("  Изчислявам word count по партия (Dask)...")
    with ProgressBar():
        word_counts = (
            ddf.groupby("party")["word_count"]
            .sum()
            .compute()  # .compute() изпълнява lazy графа
        )

    return word_counts.to_dict()


def analyze_speech_dask(speech: dict) -> dict:
    """Анализира едно изказване — използва се от Dask Bag."""
    tokens = tokenize(speech["text"])
    return {
        "party": speech["party"],
        "unigrams": dict(Counter(tokens)),
        "bigrams": dict(Counter(make_bigrams(tokens))),
        "total_words": len(tokens),
    }


def run_analysis_dask(df: pd.DataFrame, n_partitions: int = 4) -> dict:
    """
    Пълен анализ с Dask Bag.

    Dask Bag е подходящ за неструктурирани данни (списъци от речници).
    Разпределя обработката на отделни записи по партиции.
    """
    speeches = df.to_dict("records")

    print(f"  Изграждам Dask Bag с {n_partitions} партиции...")
    bag = db.from_sequence(speeches, npartitions=n_partitions)

    print("  Анализирам изказванията с Dask...")
    with ProgressBar():
        results = bag.map(analyze_speech_dask).compute()

    # Агрегирането е серийно — сливане на Counter-ите
    print("  Агрегирам по партия...")
    from analysis_serial import aggregate_by_party, compute_tfidf
    by_party = aggregate_by_party(results)
    tfidf = compute_tfidf(by_party)

    return {"by_party": by_party, "tfidf": tfidf}


if __name__ == "__main__":
    DATA_PATH = Path("data/processed/speeches.csv")
    df = load_speeches(DATA_PATH)

    print("=== Dask анализ ===")
    start = time.perf_counter()
    results = run_analysis_dask(df, n_partitions=4)
    elapsed = time.perf_counter() - start

    print(f"\nDask: {elapsed:.3f} сек.")
    print("\nИзказвания по партия:")
    for party, data in results["by_party"].items():
        print(f"  {party[:45]:45s}: {data['speech_count']:4d} изказвания, "
              f"{data['total_words']:7d} думи")
```

## 2.4 Проверка на коректността

**Задължителна стъпка преди измерванията:** всички версии трябва да дават идентични резултати. Различни времена са очаквани — различни резултати са грешка.

```python
# src/verify_correctness.py
"""
Проверява дали серийното и паралелните решения дават еднакъв изход.
Трябва да се изпълни ПРЕДИ измерванията в Фаза 3.
"""

import pandas as pd
from pathlib import Path
from common import load_speeches
from analysis_serial import run_serial
from analysis_mp import run_parallel_mp
from analysis_vectorized import run_tfidf_vectorized
from collections import Counter


def compare_top_words(serial_tfidf: dict, parallel_tfidf: dict,
                      label: str, top_n: int = 10) -> bool:
    """
    Сравнява топ N думи за всяка партия между две версии.
    Допуска малки разлики в реда при еднакви TF-IDF стойности.
    """
    ok = True
    for party in serial_tfidf:
        serial_words = set(w for w, _ in serial_tfidf[party][:top_n])
        parallel_words = set(w for w, _ in parallel_tfidf.get(party, [])[:top_n])
        overlap = len(serial_words & parallel_words) / top_n

        if overlap < 0.8:
            print(f"  ⚠ {party}: само {overlap*100:.0f}% съвпадение с {label}")
            ok = False
        else:
            print(f"  ✓ {party}: {overlap*100:.0f}% съвпадение с {label}")
    return ok


def run_verification():
    DATA_PATH = Path("data/processed/speeches.csv")

    # Зареждаме малък корпус за бърза проверка
    df = load_speeches(DATA_PATH).head(500)

    print("Стартирам серийното решение...")
    serial_results = run_serial(df)

    print("\nСтартирам multiprocessing решението...")
    mp_results = run_parallel_mp(df)

    print("\nСравнявам резултатите:")
    compare_top_words(serial_results["tfidf"], mp_results["tfidf"],
                      "multiprocessing")

    print("\n✓ Проверката е успешна — всички версии дават еквивалентни резултати")


if __name__ == "__main__":
    run_verification()
```

## Важни бележки за доклада

**Тези обяснения трябва да присъстват в доклада** — не просто „използвахме multiprocessing", а защо:

| Стъпка | Подход | Обяснение за доклада |
|--------|--------|----------------------|
| Токенизация (A) | `multiprocessing.Pool` | CPU-bound, embarrassingly parallel, заобикаля GIL |
| TF-IDF (Б) | NumPy / scipy | Data-parallel, SIMD, матрични операции в C |
| Агрегиране (В) | Dask | Lazy evaluation, работи при корпус > RAM |
| Теглене (Фаза 1) | asyncio | I/O-bound, конкурентни HTTP заявки |

**Amdahl's Law:** агрегирането по партия остава серийно. Това е Amdahl-овото ограничение на проекта — трябва да го споменеш и измериш каква му е пропорцията спрямо цялото времетраене.
